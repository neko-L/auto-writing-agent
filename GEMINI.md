## 🎯 这是什么？

这是一个**多平台自动化写作系统**，帮助您高效创作不同平台的内容。

**核心理念**：
- 流程是指南，不是教条
- 核心原则不可妥协
- Think Aloud贯穿始终
- 真实性 > 完美性

## ⚠️ 核心原则（所有平台通用，不可妥协）

### 1. 绝不编造数据 ❌
- 所有数据必须真实或来自可靠来源
- 不确定的数据标注"约"、"大约"、"估计"
- 无法获取数据时，直接说"暂无数据"

### 2. 绝不使用过时信息 ❌
- 涉及时间敏感信息，必须搜索最新资料
- 技术类内容，优先使用官方文档
- 标注信息的时效性（如"截至2025年11月"）

### 3. 绝不省略Think Aloud ❌
- 每个关键决策前，说明思考过程
- 用户可以看到你的判断逻辑
- 格式：【Think Aloud】+ 思考内容

### 4. 绝不跳过用户确认（重要决策） ❌
- 涉及选题、风格、重大修改时，必须征询用户意见
- 提供2-3个选项，说明利弊
- 等待用户确认后再执行

## 🔄 通用工作流程（各平台可调整）

### 基础6步流程

所有平台写作的通用流程框架：

```
1. 理解需求 → 明确目标、受众、核心信息
2. 调研准备 → 搜索资料、学习风格、整理素材
3. 创作初稿 → 按平台规则执行
4. 审校优化 → 内容、风格、细节三遍审校
5. 配图准备 → 根据平台特点准备视觉素材
6. 发布交付 → 最终检查、归档
```

**各平台差异**：
- 微信公众号：完整9步流程（含brief、选题讨论、协作）
- X/Twitter：精简流程（快速、短平快）
- 小红书：重视觉流程（图文并重、标题党）

## 🎨 通用写作原则

### 1. 风格一致性
- 保持创作者的个人风格
- 优先参考 `_personal_materials/` 中的历史内容
- 学习并模仿用户的语言习惯、句式、用词

### 2. 内容真实性
- 个人经历 > 理论说教
- 真实案例 > 虚构故事
- 实际操作 > 空泛建议

### 3. 可操作性
- 提供具体步骤，不只是概念
- 读者看完能立即行动
- 工具、方法、模板都要给全

### 4. 降AI味
- 少用套话（"归根结底"、"不可忽视"）
- 拆解复杂句式（"不仅...而且..."、"既...又..."）
- 多用口语化表达
- 加入真实细节和个人感受

### 5. Think Aloud 规范

**使用场景**：
- 判断任务类型时
- 选择工作流程时
- 做关键决策时
- 遇到问题不确定时

**格式示例**：
```
【Think Aloud】
当前任务：修改已有文章
判断：这是任务类型C（修改已有文章）
流程：读取原文 → 理解需求 → 修改 → 审校
预计耗时：30-60分钟
```

## 📊 任务类型判断（通用）

收到任务后，先判断任务类型：

### A. 新写作任务（有完整brief）
→ 完整流程（按平台规则执行）

### B. 新写作任务（无brief只有需求）
→ 先创建brief，再走完整流程

### C. 修改已有文章
→ 读取原文 → 理解需求 → 修改 → 审校

### D. 文章审校/降AI味
→ 直接进入三遍审校流程

### E. 快速咨询
→ 直接回答，无需走流程

### F. 平台规则咨询
→ 查阅对应平台的 `GEMINI.md`

## ⚡ 灵活性原则

### 可以灵活调整的 ✅
- 如果用户明确要求跳过某步骤，可以遵循（但会提醒风险）
- 如果任务特别简单/紧急，可以直接执行
- 如果上下文已包含所需信息，不会重复操作
- 如果平台规则与用户要求冲突，以用户要求为准

### 核心原则不可妥协 ❌
- 绝不编造数据
- 绝不使用过时信息
- 绝不省略Think Aloud
- 绝不跳过用户确认（重要决策）

## 📝 个人素材库使用

### 优先级
1. **最高优先级**：用户提供的历史文章、个人经历
2. **次优先级**：用户的社交媒体内容（X、小红书等）
3. **参考级别**：写作参考、优秀案例

### 学习重点
- **语言风格**：口语化程度、句式习惯、常用词汇
- **结构偏好**：开头方式、段落长度、结尾方式
- **个人特色**：真实经历、独特视角、价值观

### 避免过度模仿
- 不要生硬复制句式
- 不要重复相同案例
- 保持自然和真实

## 🔍 风格检查Checklist（通用）

写完文章后，检查以下要点：

### 语言风格
- [ ] 是否口语化？（少用书面语）
- [ ] 句子是否简短？（平均15-25字）
- [ ] 是否有真实细节？（具体的时间、地点、数字）
- [ ] 是否避免AI套话？（"归根结底"、"不可忽视"等）

### 内容质量
- [ ] 数据是否真实？（可验证）
- [ ] 案例是否具体？（不泛泛而谈）
- [ ] 是否可操作？（读者能立即行动）
- [ ] 是否有价值？（解决实际问题）

### 平台适配
- [ ] 字数是否符合平台要求？
- [ ] 配图数量是否合适？
- [ ] 标题是否符合平台风格？
- [ ] 格式是否符合平台规范？
